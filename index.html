    • Introduction

    Welcome to the SafeWhistle API!

    This documentation is still an early draft. To be updated.

    Authentication

    Because SafeWhistle does server-side decryption only at the moment, your network handler must exchange session cookies.

    Sessions expire after 30 minutes and will need to be either kept refreshed or re-logged in after 30 minutes of inactivity to the backend.

    All requests below require and active login to complete successfully.

    This will be resolved in a future update.

    Company Info

    Get Company Meta Information

    Returns JSON structured like this:

    {
  "statuses": [
    {
      "id": 1,
      "name": "Created",
    },
    {
      "id": 2,
      "name": "Investigating"
    }
  ],
  "categories": [
    {
      "id": 1,
      "name": "Compensation / Benefits",
    },
    {
      "id": 2,
      "name": "Discrimination"
    }
  ]
}

    Report statuses and categories have unique IDs per company. These are randomized so as to not leak information. As such you will need to grab the relationship and decrypted names for the signed-in user's company after each login.

    For the moment these are fixed and therefore cache-able for each new login.

    GET http://app.safewhistle.com/company/meta.json

    Reports

    Get All Reports

    Returns JSON structured like this:

    [
  {
    "id": "MGY2NTJmYTUtYzIwNS00MTIwLTg5MTAtZjRkNmVjMDI3M2Uz",
    "name": "I3M2Uz",
    "status_id": 1,
    "category_id": 6
  },
  {
    "id": "MTM5OTI3NjMtNDQ0OS00NThhLWJiODItYzhkODJmZGVhMzU4",
    "name": "VhMzU4",
    "status_id": 3,
    "category_id": 4
  }
]

    This endpoint retrieves all reports.

    HTTP Request

    GET http://app.safewhistle.com/company/reports.json

    Query Parameters

    Parameter Default Description
    status empty The ID of the status filtering on / "starred" for all starred reports
    category empty The ID of the category filtering on

    GET https://app.safewhistle.lo/company/reports/MWRmZmNhMDQtZjViZi00YTZkLThjZGMtMDk5Y2QzOTQ1YTUz.json

    URL Parameters

    Parameter Description
    ID The ID of the report to retrieve

    Errors

    Error Code Meaning
    401 Unauthorized -- Current session cookies are invalid.
    403 Forbidden -- The current user does not have access to the specified report.
    404 Not Found -- The specified report could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
